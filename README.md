# CBT Exam Pro

## About the Project
This project involved designing a cross-platform examination ecosystem—consisting of a Mobile App for students and a Web Dashboard for administrators. Tailored for large-scale school exams such as Mid-terms (UTS) and Finals (UAS), the system was built to democratize digital testing by providing a high-performance experience on mobile devices, ensuring that students without access to desktops could still participate in exams seamlessly.

## The Problem
* **Device Accessibility:** Many students lacked access to personal computers at home, creating a digital divide during exam periods that required a mobile-first solution.
* **Performance & Clarity:** Existing exam platforms were often too heavy for lower-end mobile devices, leading to crashes and a confusing user experience that increased student anxiety.
* **Grading Inefficiency:** Manual grading for large-scale exams was time-consuming for teachers and prone to human error, delaying the release of results and rankings.

## The Solution
I developed a "Lightweight & Focus-Oriented" interface using a unified Design System to ensure consistency across web and mobile. The UI was engineered to be clean and distraction-free, featuring an intuitive navigation system for long-form questions. Key UX enhancements included a "Pinch-to-Zoom" feature for images, a "Doubtful Answer" bookmarking system, and proactive alerts to prevent students from submitting exams with unanswered questions.

## Results
* **High Performance on Low-End Devices:** Delivered a simplified, lightweight interface that maintained stability during high-traffic exam periods, even on entry-level smartphones.
* **Immediate Feedback Loop:** Automated the scoring and ranking process, allowing students to view their performance and question discussions instantly upon completion.
* **Improved Exam Integrity:** Integrated a simulation alert system to ensure students were familiar with the platform before the actual exam, significantly reducing technical confusion and troubleshooting requests during live tests.
